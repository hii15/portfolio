# UI E2E 최소 시나리오 (운영 체크리스트)

실무에서 자주 쓰는 핵심 흐름만 빠르게 점검하기 위한 최소 시나리오입니다.

## 공통 사전 조건
- 앱 실행: `streamlit run app.py`
- Upload 탭에서 MMP 선택 후 `더미 원본 데이터 불러오기`를 먼저 수행

---

## 시나리오 1) Upload 흐름 점검
1. `데이터 업로드` 탭 진입
2. `더미 시나리오 선택` 기본값 유지 후 더미 로드
3. 설치/이벤트/비용 미리보기 표가 나타나는지 확인
4. 아래 항목 다운로드 버튼 클릭 가능 여부 확인
   - 설치 원본 CSV
   - 이벤트 원본 CSV
   - 비용 원본 CSV
5. 템플릿 다운로드 버튼 클릭 가능 여부 확인
   - 설치 템플릿 CSV
   - 이벤트 템플릿 CSV
   - 비용 템플릿 CSV

성공 기준
- 더미 로드 성공 메시지가 노출됨
- 표 3개(설치/이벤트/비용) 렌더링
- 다운로드 버튼 클릭 시 파일 응답이 내려옴

---

## 시나리오 2) Decision 탭 점검
1. `UA 판단` 탭 진입
2. 판단 기준 레벨을 `캠페인+광고그룹`으로 변경
3. 판단 결과 요약(증액/감액/보류/유지) 카드 확인
4. `감액/보류 우선 정렬` on/off 동작 확인
5. 판단 결과 표에 아래 컬럼이 보이는지 확인
   - 판단
   - 판단 사유
   - 효율 상태
   - 목표 대비 ROAS 차이(%)
   - 최소 설치수 대비 차이

성공 기준
- 레벨 변경 시 오류 없이 표와 요약 카드가 갱신됨

---

## 시나리오 3) LiveOps 탭 점검
1. `라이브옵스 영향` 탭 진입
2. 비교 기준 레벨을 `캠페인+광고그룹`으로 변경
3. 기본 기간에서 상/하위 카드 및 요약 테이블 렌더링 확인
4. 데이터가 거의 없는 기간으로 변경해 빈 결과 안내문 확인

성공 기준
- 정상 기간: 카드/테이블 표시
- 빈 기간: 에러 크래시 없이 안내문 표시

---

## 시나리오 4) Experiment 탭 점검
1. `실험 리포트` 탭 진입
2. `MMP 실험 실행` 클릭
3. 인사이트 카드(최고/최저/변동성) 확인
4. 다운로드 버튼 동작 확인
   - 요약 CSV
   - 판단결과 CSV
   - 인사이트 리포트(MD)

성공 기준
- 실행 성공 메시지 노출
- 카드/표/리포트 렌더링
- 다운로드 버튼 모두 정상 동작

---

## 권장 운영 방식
- 배포 전 최소 1회 위 4개 시나리오 점검
- 회귀 이슈가 생기면 재현 시나리오 번호를 이 문서 기준으로 기록
